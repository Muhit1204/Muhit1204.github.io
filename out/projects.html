Starlink Maritime Performance Prediction & Decision-Support Dashboard

Ongoing Project
  • —Developed a real-time, predictive decision-support web application for maritime Starlink users, ingesting live network telemetry to forecast throughput and latency up to 15 minutes ahead.
  • —Predictive Forecasting & Confidence Intervals: Implemented a forecasting engine (based on Random Forest regression logic) that projects network throughput and latency across a 15-minute horizon, visualizing the trend curve alongside dynamic confidence intervals that expand with time.
  • —Early Warning & Anomaly Detection: Engineered an analytical system that monitors the relationship between latency and throughput to preemptively detect satellite handoffs and issue early warnings when rising latency indicates an impending drop in bandwidth.
  • —Actionable Decision Support: Designed a "Task Readiness" algorithm that translates raw network forecasts into practical recommendations, calculating the exact "Safe Duration" (in minutes) for specific operations like Video Calling, VoIP, and Large File Transfers.
  • —Real-Time Accuracy Tracking: Built a continuous feedback loop that stores short-term predictions and evaluates them against actual incoming telemetry, displaying a live Exponential Moving Average (EMA) of the model's accuracy.
  • —Live Data & Contextual Integration: Developed features for both live telemetry simulation and historical CSV data ingestion, alongside a REST API integration (Open-Meteo) to fetch real-time weather data for environmental context.
  • —Interactive Data Visualization: Created a responsive, dark-themed maritime UI using Next.js and Tailwind CSS, featuring complex Recharts visualizations that seamlessly combine historical data, future trend lines, and shaded uncertainty ranges.
